glz_custom_fields, limitless pimped custom fields

If you feel totally lost, there is a screencast showing the basics of this plugin: glz_custom_fields_unveiled.mov. The post associated with this screencast is here: glz_custom_fields unveiled.

Requirements

There is a single requirement now, my modified version of TXP 4.0.8.

If you want to use the special custom set type Date Picker, you will need to place the jquery.datePicker folder in a scripts folder at the root of your website. If your textpattern folder is /var/www/mysite/textpattern, the jquery.datePicker folder will be /var/www/mysite/scripts/jquery.datePicker. The jquery.datePicker folder will come archived, as an attachment alongside the plugin as of v1.2.1.

Usage and Options

Because this is a back-end plugin primarily, there is only 1 front-end tag:

<txp:glz_custom_fields_search_form results_page="#" searchby="Area:select, City:radio, Price:checkbox, section, category" section="articles" />
  • searchby (must be set) - which custom fields should we search by and in what order? Also provides capability to search by section and category, they default to checkboxes and cannot be changed. In the above example, we are displaying both the section & category select boxes, if no section is selected, it will default to *articles* section ONLY. If you want to search in all sections by default, do section="all".
  • results_page - which page should the submit button take the user to. In our example this will send the POST request to the same page - will become available as of v1.2.x. (search by default).

Tricks

Actually, there is a single one.

If you want to define ranges, create a new custom field with the following name e.g. Price range &pound;(after)

  • range - this is really important and must be present for the range to work correctly
  • &pound; - don't use straight symbols (like € or £) but entity ones e.g. &pound; & &euro; $ and &cent are valid symbols.
  • (after) - don't leave any spaces after e.g. $ and (after). This basically says where to add the measuring unit (be it a currency or something like m3 etc.) - (before) is default.

Ranges are defined 10-20, 21-30 etc. (no measuring units - they get pulled from the custom set name).

Colophon & Limitations

PLEASE back-up your database before using this plugin. It won't delete anything automatically, but if something goes wrong, you might end up deleting all your custom fields data (and I really mean all of it).

If you don't have JavaScript enabled in your browser, things will blow up - GUARANTEED.
